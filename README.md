# Coverage utility

## Назначение
coverage.py  - это скрипт, который автоматически собирает CMake или make-проект с поддержкой анализа покрытия тестами и формирует HTML-отчёт с результатами. Он облегчает процесс измерения покрытия кода тестами при помощи инструментов gcov, lcov, genhtml.

Основные возможности:
- добавление необходимых флагов компиляций
- сборка проектов через cmake/make
- запуск тестов через ctest
- создание HTML-отчётов о покрытие кода

## Системные требования 

- Операционная система: Astra Linux, Ubuntu, Debian или другой Linux-дистрибутив.
- Python: версия 3.6 или новее
- Компилятор: версия GCC/G++ 9.0 или новее
- CMake: версия 3.16 или новее

## Запуск 

1. Скопируйте файл coverage.py в каталог проекта
2. Перейдите в каталог проекта
cd <имя_вашего_проекта>
3. Сделайте скрипт исполняемым:
chmod +x coverage.py
4. Запустите:
./coverage.py

Если всё прошло успешно, откроется HTML-страница index.html с визуальным отображением. 

## Как это работает

1. Скрипт проверяет наличие CMakeLists.txt
2. Добавляет недостающие флаги —coverage
3. Собирает проект в режиме Debug
4. Запускает тесты через ctest
5. Сохраняет результаты в coverage.info и создаёт HTML-отчёт

## Опции

Скрипт поддерживает ряд аргументов командной строки, которые управляют его поведением. Посмотреть полный список и описание можно с помощью встроенной справки:

./coverage.py --help

Примеры параметров:
./coverage.py --no-open (не открывать HTML-отчёт в браузере)
./coverage.py --no-install (не устанавливать зависимости через apt-get)
./coverage.py --fix-cmake (автоматически добавить недостающие строки в CmakeLists.txt)
./coverage.py --no-clean (не удалять каталог сборки перед конфигурацией)
